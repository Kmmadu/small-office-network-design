# Contributing to Small Office Network Design

Thank you for considering contributing to this project! This document provides guidelines for contributing.

## How to Contribute

### Reporting Issues
- Check if the issue already exists
- Provide clear description of the problem
- Include Packet Tracer version information
- Attach screenshots if applicable

### Suggesting Enhancements
- Clearly describe the enhancement
- Explain why it would be useful
- Provide examples if possible

### Contributing Code/Configurations
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test thoroughly in Packet Tracer
5. Commit your changes (`git commit -m 'Add amazing feature'`)
6. Push to the branch (`git push origin feature/amazing-feature`)
7. Open a Pull Request

## Guidelines

### Network Design Changes
- Follow Cisco best practices
- Document all configuration changes
- Update relevant documentation files
- Ensure backward compatibility when possible
- Test all changes thoroughly

### Documentation
- Use clear, concise language
- Follow existing documentation style
- Update README.md if adding major features
- Include examples where helpful

### Configuration Standards
- Use consistent naming conventions
- Comment complex configurations
- Follow security best practices
- Document any deviation from standard configurations

## Testing Requirements

Before submitting changes:
- [ ] All connectivity tests pass
- [ ] Security features work as intended
- [ ] DHCP assigns correct addresses
- [ ] NAT/PAT functions properly
- [ ] ACLs block appropriate traffic
- [ ] Documentation is updated

## Code of Conduct

### Our Standards
- Be respectful and inclusive
- Accept constructive criticism
- Focus on what's best for the community
- Show empathy towards others

### Unacceptable Behavior
- Harassment or discriminatory language
- Personal attacks
- Publishing others' private information
- Other unprofessional conduct

## Questions?

Feel free to open an issue for questions about contributing.

## Recognition

Contributors will be acknowledged in the project documentation.

Thank you for helping improve this project!
